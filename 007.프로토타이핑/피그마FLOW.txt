[ 피그마 FLOW ]

1. 프로토타입 사용시 각 페이지 흐름을 
    구분하기 위해 사용하는 라벨

2. Prototype 메뉴에서 flow starting position메뉴의 +버튼을 클릭하여 생성함
    (-버튼으로 변경시 클릭하면 지워짐)

3. 목적은 여러 흐름의 사이트를 생성시 
    특정 파트에 대한 구분을 위해 라벨 개념으로 붙이는 것이다.

4. Flow 라벨 옆에 플레이버튼이 있어서 바로 그 곳 부터 브라우저 구동하는 서비스도 
    제공함

5. 제작된 컴포넌트는 Assets 메뉴에 있다.
    -> 또는 상단 메뉴에 REsources 버튼 아래에도 컴포넌트를 볼 수 있다.

6. 컴포넌트 내부에 컴포넌트를 묶어서 작업할 경우
    내부 컴포넌트를 Variant(변형)라고 부른다.
    -> 단어의 뜻을 볼때 기본형으로부터 변경시에 사용된다.

    
    
    
    
    [ 피그마 컴포넌트 배너1 ]

1. 박스만들고 이미지넣기
2. 컴포넌트로 변경
3. Variants 버튼으로 내부 컴포넌트 만들기
4. 반복하여 4개 생성
5. Prototype 메뉴에서 각 내부 컴포넌트를 아래로 연결
6. 각 속성 이벤트는 After Delay 를 선택
    (여기서 시간은 1/1000초임 - 2000ms는 2초)
7. 애니메이션 옵션은 Dissolve 선택
    (이징가속도 : ease-in-out 처음과 끝에 천천히)
8. Assets 또는 Resources에서 해당 컴포넌트를
    드래그 하여 원하는 프레임에 넣기
9. Present 화면 띄우기(ctrl+alt+엔터)

※ 만약 배너가 안움직이면 로딩시간이 안맞아서
그럴 수 있으므로 오른쪽 아래 Restart 버튼을 클릭한다!
